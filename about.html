About MD Reader

Built for a simple reason: you get sent a .md file on your phone, and you just want to read it — without installing an app or uploading it to someone else's server.

What it does

  • Open local files: tap or drop a .md / .markdown / .txt file and see it rendered instantly.
  • Import from a URL: paste a raw Markdown link (e.g. a GitHub raw URL) and open it.
  • Export: save as PDF, download as .html, or copy as rich text to paste into Word, email, etc.
  • Toggle views: switch between rendered output and raw source anytime.
  • Install as an app: add to your home screen for offline access, just like a native app.

How to add to your home screen

Once added, you will see an icon on your home screen. Tap it to open the reader — just like a native app, even offline. Instructions by platform:

🟢 HarmonyOS

Open this page in your browser → tap the menu (bottom or top-right) → select "Add to home screen" → confirm.

🤖 Android

Open in Chrome or your default browser → tap the menu "⋮" → "Add to Home Screen" / "Install app" → confirm.

🍎 iPhone / iPad

You must use Safari. Open this page → tap the Share button (box with arrow) → "Add to Home Screen" → Add.

💻 Windows / Mac

Open in Chrome or Edge → click the Install icon in the address bar (screen with down arrow) → Install. You can also find "Install this app" in the browser menu.

💡 If you see an "+ Add to Home Screen" button at the top of the homepage, just tap that.

Why no-upload matters

All parsing and rendering happens right in your browser. Your files are never sent to any server. This means: faster (no upload wait), more private (contracts, notes, documents stay on your device), and works offline. You can verify this — open browser DevTools and check the Network tab: after the page loads, there are zero upload requests.

Supported Markdown syntax

Full GitHub-Flavored Markdown (GFM) support: headings, lists, tables, task lists, strikethrough, blockquotes, links, images, fenced code blocks with language detection, and inline HTML. Renders just like GitHub or Notion.

FAQ

Are my files uploaded?

No. All processing is done locally in your browser with marked.js. Your files never leave your device.

Does it work offline?

Yes. The first visit caches the app locally. After that you can open local .md files even without internet. Installing to your home screen gives the best offline experience.

How is this different from the built-in text viewer?

Your phone's text viewer shows raw markup (# Title, **bold**) — hard to read. This tool renders it into proper headings, bold text, tables, and code blocks, just like you'd see on GitHub.

Why does URL import fail?

You need to use a raw Markdown URL, e.g. raw.githubusercontent.com/…, not a regular web page. Some sites block cross-origin requests — if that happens, download the file and open it locally.

Is it free? Do I need an account?

Completely free. No login, no signup, no account system at all.

Tech

Static site — HTML + CSS + vanilla JS, no backend, no database, no build step. Markdown parsing by marked.js. Local file reading via the browser File API. Offline support via Service Worker. Installation via Web App Manifest (PWA). No tracking, no third-party analytics.
