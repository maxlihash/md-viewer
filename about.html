关于 MD 阅读器

做这个工具的原因很简单：手机上收到一个 .md 文件，想看一眼，不该还要专门装个 App、还要把文件传到别人服务器上。

它能做什么

  • 打开本地文件：点一下或拖进来，.md / .markdown / .txt 立刻排版成网页。
  • 链接导入：粘贴一个 Markdown 文件的直链（如 GitHub raw 链接），点「打开」即可。
  • 导出：可导出 PDF、存为网页文件、一键复制（带格式，能直接粘进微信/飞书/Word）。
  • 源码切换：随时在「排版后」和「原始 Markdown」之间切换。
  • 添加到桌面：像装 App 一样放到桌面，离线也能用。

怎么添加到桌面

添加后，桌面会多一个图标，点开就是这个工具，跟原生 App 一样，断网也能打开本地文件。各系统的步骤：

🟢 鸿蒙 HarmonyOS

浏览器打开本页 → 点底部/右上角菜单 → 选「添加到桌面」/「添加到主屏幕」→ 确认。

🤖 安卓 Android

用 Chrome / 自带浏览器打开 → 右上角菜单「⋮」→「添加到主屏幕」/「安装应用」→ 确认。

🍎 iPhone / iPad

必须用 Safari 打开 → 点底部「分享」按钮（方框+箭头）→「添加到主屏幕」→ 添加。

💻 电脑 Windows / Mac

用 Chrome / Edge 打开 → 地址栏右侧的「安装」图标（屏幕带下箭头）→ 安装。也可在浏览器菜单里找「安装此应用」。

💡 如果首页右上角出现了「＋ 添加到桌面」按钮，直接点它最省事。

为什么免上传更好

所有解析、排版都在你自己的浏览器里完成，文件不会发到任何服务器。这意味着：更快（没有上传等待）、更私密（公司文档、合同、笔记不外泄）、断网也能用。打开浏览器的开发者工具看网络请求就能验证——页面加载完之后，看文件不会再产生任何上传请求。

支持哪些 Markdown 语法

完整支持 GitHub 风格 Markdown（GFM）：标题、列表、表格、任务清单、删除线、引用、链接、图片、代码块（带语言识别）、行内 HTML。排版效果和 GitHub、Notion 上看到的基本一致。

常见问题

我的文件会被上传吗？

不会。全部处理都在浏览器本地用 marked.js 完成，文件不发往任何服务器。

断网能用吗？

能。第一次访问后会缓存到本地，之后离线也能打开本地 .md 文件。添加到桌面后体验更接近原生 App。

和手机自带的「文本查看」有什么区别？

自带查看器显示的是原始符号（# 标题、**加粗**），看着乱。这个工具会把它排版成真正的标题、加粗、表格、代码块，跟在 GitHub 上看一样。

链接导入打不开 / 报错？

需要是 Markdown 文件的直链（raw），比如 raw.githubusercontent.com/...，不是网页地址。部分站点限制跨域访问也会失败，这种情况把文件下载到本地再打开即可。

收费吗？要登录吗？

完全免费，不用登录、不用注册，没有任何账号体系。

技术说明

纯静态页面，前端 HTML + CSS + 原生 JS，没有后端、没有数据库、没有打包步骤。Markdown 解析用 marked.js，本地文件读取用浏览器 File API，离线用 Service Worker 缓存，添加到桌面靠 Web App Manifest（PWA）。无埋点、无第三方统计。
